# SimTradeLab v5.0 插件化架构实施TODO计划

## 📋 总体说明

本文档是基于v4.0完整架构设计和v5.0新增特性的分阶段实施计划。严格按照"100%测试覆盖"原则执行：每个阶段必须达到100%测试覆盖率后才能进入下一阶段。

v5.0在v4.0基础上聚焦于提升系统的健壮性、开发效率和可维护性，新增了事件总线标准化、可插拔回测组件、配置验证机制、统一测试框架、插件依赖管理和企业级多租户支持。

## 🎯 实施原则

1. **测试驱动开发**：每个功能都要先写测试，再写实现
2. **100%覆盖要求**：单元测试、集成测试、边界测试全覆盖
3. **阶段性验证**：每个阶段完成后进行完整的功能验证
4. **向后兼容**：确保每个阶段都不破坏已有功能
5. **渐进式交付**：每个阶段都能独立运行和测试
6. **健壮性优先**：v5.0重点关注系统稳定性和可维护性

## 🚨 架构错误修复清单 (紧急优先级)

### 错误分析说明
基于对当前实现与SimTradeLab v5.0架构设计文档的对比分析，发现以下违反设计原则的实现错误。这些错误需要在进行后续功能开发之前优先修复，以确保架构的正确性和一致性。

### 🔥 高优先级架构错误 (必须立即修复)

#### E1. PTrade适配器架构分层错误
- **错误描述**: PTrade适配器被错误实现为插件(`BasePlugin`子类)，违反了v5.0三层架构设计
- **违反原则**: 
  - v5.0架构明确定义PTrade适配器属于"平台适配层"
  - 适配器应该是核心框架的一部分，不是插件
- **影响范围**: 核心架构分层混乱
- **修复方案**: 
  - 重构`PTradeAdapter`为独立的适配器类，不继承`BasePlugin`
  - 创建专门的适配器基类`BaseAdapter`
  - 更新Runner中的适配器使用方式
- **文件影响**: 
  - `src/simtradelab/adapters/ptrade/adapter.py`
  - `src/simtradelab/adapters/base.py` (新建)
  - `src/simtradelab/runner.py`
- **测试要求**: 确保PTrade适配器功能不受影响
- **状态**: **✅ 已修复**

#### E2. Runner依赖关系设计错误  
- **错误描述**: Runner直接管理PluginManager和PTradeAdapter，违反依赖倒置原则
- **违反原则**:
  - Runner作为用户接口层不应直接管理核心组件
  - 应该通过核心引擎或工厂模式管理依赖关系
- **影响范围**: 系统耦合度过高，扩展性差
- **修复方案**:
  - 创建`CoreEngine`类作为核心引擎
  - Runner通过CoreEngine间接使用底层组件
  - 实现依赖注入机制
- **文件影响**:
  - `src/simtradelab/runner.py`
  - `src/simtradelab/core/engine.py` (新建)
- **测试要求**: 确保现有Runner API兼容
- **状态**: **✅ 已修复**

#### E3. 数据插件发现性能问题
- **错误描述**: PTrade适配器在交易路径中进行运行时插件发现，违反性能优先原则
- **违反原则**:
  - v5.0明确"性能优先：核心交易功能不使用插件，确保最佳性能"
  - 运行时发现增加交易延迟
- **影响范围**: 交易性能下降
- **修复方案**:
  - 在初始化阶段完成所有插件发现和缓存
  - 交易阶段只使用预先解析的插件引用
  - 实现插件预加载和缓存机制
- **文件影响**:
  - `src/simtradelab/adapters/ptrade/adapter.py`
  - `src/simtradelab/core/plugin_manager.py`
- **测试要求**: 验证性能提升效果
- **状态**: **✅ 已修复**

### 🔧 中优先级架构错误 (短期内修复)

#### E4. 自动注册机制复杂性问题
- **错误描述**: PluginManager的自动发现机制过于复杂，违反显式注册原则
- **违反原则**:
  - v5.0设计倾向于显式的插件注册
  - 当前AUTO_REGISTER机制增加了系统复杂性
- **影响范围**: 插件管理复杂性增加
- **修复方案**:
  - 简化为显式注册机制
  - 保留自动发现作为可选功能
  - 提供更清晰的插件注册API
- **文件影响**:
  - `src/simtradelab/core/plugin_manager.py`
  - 所有插件文件中的AUTO_REGISTER配置
- **测试要求**: 确保插件注册功能正常
- **状态**: **✅ 已修复**

#### E5. API路由器职责混乱
- **错误描述**: API路由器包含业务逻辑，违反单一职责原则
- **违反原则**:

  - 业务逻辑应该在独立的服务类中
- **影响范围**: 代码可维护性降低
- **修复方案**:
